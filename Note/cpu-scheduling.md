## CPU 스케줄링이란
- 운영체제가 프로세스들에게 공정하고 합리적으로 CPU 자원을 배분하는것
- 빨리 처리해야 하는 프로세스가 있기 때문에 필요 (입출력이 많은 프로세스, CPU 작업이 많은 프로세스)

## 프로세스 우선 순위
- 프로세스 PCB에 저장된다

### 스케줄링 큐
- 운영체제가 일일히 프로세스 PCB를 확인하며 우선순위를 정하는 것은 매우 비효율적이기 때문에 생겼다
- 특정 자원(예를 들면 CPU, 하드디스크, 프린터등)을 사용하고 싶어하는 프로세스들끼리 줄을 세워 관리한다 
- 우선순위가 우선인 프로세스부터 실행됨
- 준비 큐와 대기 큐가 있다
- 입출력 장치 -> 대기 큐 -> 준비 큐 -> CPU 순서 (항상 일정하지는 않음)
- 입출력 장치가 필요없는 프로세스들은 바로 준비 큐에 들어가게 된다

#### 준비 큐
- CPU를 위해 기다리는 프로세스들을 순서대로 모아놓은  큐

#### 대기 큐
- 입출력장치를 이용하기 위해 기다리는 프로세스들을 순서대로 모아놓은 큐

### 선점형 스케줄링
- 프로세스 하나가 CPU를 독점하지 못함
- 돌아가면서 정해진 시간 만큼만 사용하게 만든 스케줄링(타이머 인터럽트)
- 프로세스들에게 골구로 자원을 배분할 수 있다
- 문맥교환이 자주 일어나 오버헤드가 생길수 있다

### 비선점형 스케줄링
- CPU를 사용하고 있는 프로세스가 작업이 완전히 다끝날때까지 다음 프로세스가 CPU를 사용하지 못한다
- 선점형의 정 반대 개념 

## 스케줄링 알고리즘


### 선입 선처리 스케줄링
- FCFS(First Come First Served) 스케줄링
- 단순히 준비큐에 삽입된 순서대로 처리하는 비 선점형 스케줄링
- 먼저 CPU를 요청한 프로세스부터 CPU를 할당
- 프로세스들이 기다리는 시간이 매우 길어질 수 있다는 부작용이 있다 (호위 효과)

### 최단 작업 우선 스케줄링
- SJF(Shortest Job First) 스케줄링
- 호위 효과를 방지
- CPU 사용이 긴 프로세스는 나중에 실행, CPU 사용 시간이 짧은 프로세스는 먼저 실행
- CPU 사용 시간이 가장 짧은 프로세스부터 처리하는 스케줄링 방식

### 라운드 로빈 스케줄링
- RR(Round Robin) 스케줄링
- 선입 선처리 스케줄링 + 타임 슬라이스(time slice)
- 타임 슬라이스란 각 프로세스가 CPU를 사용할 수 있는 정해진 시간
- 정해진 타임 슬라이스 만큼의 시간 종안 돌아가며 CPU를 이용하는 선점형 스케줄링
- 타임 슬라이스 크기가 중요하다

### 최소 잔여 시간 스케줄링
- SRT(Shortest Remaining Time) 스케줄링
- 최단 작업 우선 스케줄링 + 라운드 로빈 스케줄링
- 최단 작업 우선 스케줄링 : 작업 시간이 짧은 프로세스부터 처리하는 스케줄링 알고리즘
- 라운드 로빈 스케줄링 : 정해진 타임 슬라이스만큼 돌아가며 사용하는 스케줄링 알고리즘
- 정해진 시간만큼 CPU를 이용하되, 다음으로 CPU를 사용할 프로세스로는 남은 작업 시간이 가장 적은 프로세스 선택

### 우선 순위 스케줄링
- 프로세스들에 우선순위를 부여하고, 우선 순위 높은 프로세스부터 실행
- 우선 순위가 같은 프로세스들은 선입 선처리로 스케줄링
- 최단 작업 우선 스케줄링은 작업 시간이 짧은 프로세스에게 높은 우선순위를 , 최소 잔여 시간 스케줄링은 남은 작업 시간이 짧은 프로세스에게 높은 우선순위를 주기 때문에 둘다 우선 순위 스케줄링이라고 할 수 있다
- 기아 현상이 생길 수 있다
- 기아(starvation) 현상: 준비 큐에 먼저 삽입되었음에도 불구하고 우선순위가 계속 밀려 실행이 연기되는 현상
- 이를 방지하기 위한 기법으로는 에이징(aging) 기법이있다
- 오랫동안 대기한 프로세스의 우선순위를 점차 높이는 방식 (마치 나이를 먹는다고해서 에이징)  

### 다단계 스케줄링
- Multilevel queue 스케줄링
- 우선 순위 스케줄링의 발전된 형태
- 우선 순위 별로 준비 큐를 여러개 사용하는 스케줄링 방식

### 다단계 피드백 큐 스케줄링
- Multilevel feedback queue 스케줄링
- 다단계 큐 스케줄링의 발전된 형태
- 큐 간의 이동이 가능한 다단계 큐 스케줄링
- 다단계 큐 스케줄링에서는 기본적으로 큐 간의 이동이 불가 (이는 기아현상을 만들 수 있음)
